Compententies

1. ANALYSEREN:
Deze compententie heb ik sterk ontwikkeld bij PEE20. Hierbij was de uitdaging om de opdracht om te zetten naar ontwerpeisen. In het begin had ik daar veel moeite mee. Na een paar keer proberen met feedback kan ik uiteindelijk een opdracht om zetten naar ontwerpeisen en deze verdelen in functionele- en niet-functionele eisen. Hierbij gaat het niet altijd helemaal vlekkeloos en streef ik ook er naar om dit te verbeteren in de toekomst.

2. ONTWERPEN:
Voor PEE10 moest ik een equalizer ontwerpen. De equalizer had bepaalde eisen waaraan deze moest voldoen. Deze eisen zijn al vastgesteld bij het onderzoeken. Ook bij het onderzoeken zijn er naar potentiele circuits gekeken en daarbij één circuit uit gekomen. Dit circuit moeten we echter aanpassen om aan de eisen te kunnen voldoen. Hierbij was het nodig om de weerstand en condensatoren waardes uit te rekenen voor elke filter en deze toe te passen in het circuit.
Vervolgens kunnen we deze circuit simuleren d.m.v. een programma namelijk LTspice. Ook testen we vervolgens dit circuit door het op een breadboard te maken en deze aan te sluiten aan de functiegenerator en oscilloscoop.
Als de schakeling correct functioneert kan deze vervolgens op een developmentboard of PCB gesoldeerd worden. Hierbij heb ik eerst een simpele tekening gemaakt hoe de verschillende componenten op een developmentboard of PCB passen. Hierdoor kun je componenten verkeerd solderen voorkomen en zorgt het ook voor meer overzicht.
Als het circuit niet werkt dan onderzoeken we waar het fout gaat in de schakeling d.m.v. meten met oscilloscoop en dit te vergelijken met de simulaties. Hierbij is het ook handig om rond te vragen voor hulp om het probleem zo efficient mogelijk op te kunnen lossen. Eventueel moet er gezocht worden naar een nieuwe schakeling of één zelf ontwerpen, waarna je door weer alle stappen door heen gaat.
Ook is het nodig om de UI van de equalizer te ontwerpen zodat de gebruiker de equalizer kan gebruiken. Dit was simpel gedaan door aan te geven wanneer een potmeter op 0% of 100% stond. Hierbij werd er gekeken naar wat het meest logische manier was om de gebruiker feebback te geven wanneer hij iets instelt.

4. BEHEREN:
Bij PEE20 was er hardware en software. Bij het gedeelte software moest je je code bijhouden. Dit kon je doen met een programma genaamd BitBucket. Hiermee kan je je code uploaden naar een cloud waar je code vervolgens wordt opgeslagen. Elke keer wanneer er nieuw stuk code aanwezig is kan je deze nieuw stuk code committen naar de cloud waar vevolgens het programma zelf een "tijdlijn" aanmaakt. Hiermee kun je zien hoe je code zich in de loop der tijd heeft ontwikkeld. Door dit telkens te doen beheer je je code en kunnen anderen ook zien hoe je code is ontstaan of waar het mis ging.
(PS op deze manier is ook deze website gebouwd maar dan met GitHub)

6. ONDERZOEKEN:
Voor PEE10 kreeg ik de opdracht van mijn team om een equalizer te bouwen. Uit de requirement van de equalizer kwamen de volgende eisen:
~ De situatie hierbij is dat we een equalizer willen maken met drie frequentiebanden, namelijk: laag, midden en hoog;
~ De frequentieband laag regelt alle frequenties van 0 t/m 1 kHz;
~ De frequentieband midden regelt alle frequenties van 1 t/m 4 kHz;
~ De frequentieband hoog regelt alle frequenties van 4 t/m ∞ kHz;
~ Hierbij willen wij de verschillende banden hoog of laag zetten.

Hieruit kon ik concluderen dat ik een 3 bands equalizer moest maken. Er zijn op internet en in boeken veel circuits te vinden. Hierbij vond ik de volgende eisen belangrijk bij een circuit. Hij is simpel, niet te veel componenten dan nodig, lage kosten en is te verbeteren als het nodig is. Met deze eisen ging ik zoeken naar circuits die aan deze eisen voldeden. Verder heb ik ook gekeken naar circuits over filters die ik bij ELE10 heb behandeld. Daarbij kwam ik uit op één schakeling die voldeet aan alle eisen en die ik ook begreep om nodige aanpassingen te verrichten.
